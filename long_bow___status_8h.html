Enumeration Type Documentation

enum LongBowStatus

The status of an individual Test Case, and aggregate Test Fixture, or Test Runner.

Status is either successful or not and each has a subset of qualifiers.

A successful status is an outright success, or a qualified success.

Qualified success is a test that issued an explicit warning (via testWarn(...)), was purposefully skipped (via testSkip(...)), or is unimplemented (via testUnimplemented(...)).

Correspondingly, an unsuccessful test is outright failure, or a qualified failure. Qualified values indicate that the test process received a signal during the test, or either the Test Fixture set-up or tear-down steps signaled failure.

Enumerator
LongBowStatus_DONTCARE 

Used for expressing the expected status.

LongBowStatus_UNTESTED 

The test was not run (initial state).

LONGBOW_STATUS_SUCCEEDED 

The test was successful.

LongBowStatus_WARNED 

The test was successful, but with a warning.

LongBowStatus_TEARDOWN_WARNED 

The test was successful, but the tear down issued a warning.

LONGBOW_STATUS_SKIPPED 

The test was purposefully skipped by the test implementor.

LongBowStatus_UNIMPLEMENTED 

The test was incomplete because it signals that it is not implemented.

LongBowStatus_IMPOTENT 

The test ran but evaluated nothing.

LONGBOW_STATUS_SETUP_SKIPTESTS 

The setup function signals that all of the subordinate test cases must be skipped.

LONGBOW_STATUS_FAILED 

The tests failed.

LongBowStatus_STOPPED 

The test failed because it was stopped by a signal.

LONGBOW_STATUS_TEARDOWN_FAILED 

The tear down of the test failed.

This doesn't imply that the test failed.

LONGBOW_STATUS_SETUP_FAILED 

The test was incomplete because the setup for the test failed.

LONGBOW_STATUS_MEMORYLEAK 

The test was incomplete because a memory leak was detected.

LongBowStatus_SIGNALLED 

The test failed due to an uncaught signal.

LongBowStatus_LIMIT 

The limit of LongBowStatus values
